## 0. 合规与免责边界（前置必读）
**为规避金融合规风险，本项目全域采用以下硬性约束：**
1. **话术修正**：彻底摒弃“资金托管”词汇，统一采用**「密码学资金锁定调度」**。AEP 协议不碰触、不沉淀资金，仅通过 CAW MPC 调度资金状态的流转。
2. **资产边界**：全域运行于 Base Sepolia 测试网，**绝无真实金融资产服务、无募资行为**。
3. **免责声明**：前端大屏首屏、PPT首页必须展示醒目免责：`⚠️ 本项目为技术协议演示，仅使用测试网资产，无真实金融服务。`

## 4. 部署架构与环境策略
### 4.1 Docker Compose 极简部署
- **镜像构建**：在 `docker-compose.yml` 中固定基础镜像版本（如 `golang:1.22-alpine`），配置资源限制（`mem_limit: 512m`），防止单点OOM拖垮演示机。
- **敏感信息**：摒弃硬编码，CAW API Key、Relayer私钥等通过 `.env` 文件注入，`docker-compose.yml` 使用 `environment` 引用。
### 4.2 离线预案与切换流程 (防断网翻车)
- **数据源一致性**：离线 JSONL 文件**严禁手写Mock**，必须通过 Foundry的 `cast` 命令从测试网真实抓取事件导出，保证在线/离线数据结构100%统一。
- **一键切换SOP**：
  1. 检测网络：脚本 `deploy_demo.sh` 执行 `curl -m 5 <RPC_URL>`。
  2. 切换：若失败，自动在 `.env` 写入 `MODE=offline` 并重启容器。
  3. 验证：后端启动打印 `[Startup] Running in OFFLINE mode`，前端连接状态灯显示 🟡离线模式。

## 6. 功能模块设计 (实战级防翻车强化)
### F1: 悬赏发布与密码学锁定
- **全局业务ID幂等**：`BizID = SHA256(BuyerAddr + BlockNumber + Nonce)`。调用 CAW 创建 Pact 时传入 `BizID`，防止网络重试导致资金重复锁定。
- **超时退款逻辑**：Pact 创建时写入超时区块条件，合约提供 `refundAfterTimeout` 兜底函数，任何人可调用，杜绝资金永久锁死。
### F2: 身份准入与抢单 (防死锁与可插拔)
- **FluxA 可插拔降级**：
  - 配置项 `fluxa.enabled`，若关闭则直接允许抢单。
  - 若开启，使用 `context.WithTimeout(ctx, 3*time.Second)`，超时直接降级放行，记录 `FallbackUsed=true`。
- **Redis SETNX 防死锁**：
  1. 锁 TTL 设置为任务最大时长的 2 倍（兜底防死锁）。
  2. **必须使用 `defer redis.Del()`**，无论协程是否 panic，强制释放锁。
- **PG 行锁二次兜底**：获取 Redis 锁后，执行 `SELECT FOR UPDATE` 校验状态，彻底杜绝并发重复抢单。
### F3: 双轨校验与裁决 (LLM容错降级)
- **RuleEvaluator**：执行 Go 硬逻辑。返回 `Pass` / `Fail`。
- **LLM Evaluator**：调用 OpenAI SDK，严格校验返回的 JSON 格式。
  - **格式校验失败 / 超时 / 命中注入敏感词 ➡️ 直接降级忽略 LLM 结果**，仅以规则主轨为唯一裁决依据。
- **Aggregator 聚合规则**：
  - 规则 `Fail` ➡️ **直接退款** (无视 LLM)
  - 规则 `Pass` + LLM `HighRisk` ➡️ **进入 PendingReview 人工审核** 🟡
  - 规则 `Pass` + LLM 正常/降级忽略 ➡️ **正常结算** 🟢
### F4: 条件结算 / 风险退款 (防权限失控与防重复)
- **🚀 致命漏洞修复：用户确认兜底机制**
  - **问题**：Relayer 自动结算权限过大，评委必质疑“AI乱划钱怎么办”。
  - **修复**：前端增加 **「Buyer 模拟确认结算」** 按钮。Relayer 执行 CAW Release 前，**必须校验后端存在该 Job 的 `BuyerApproval = true` 记录**。
  - **日志显性打印**：`trace_id:xxx, job_id:xxx, event:CAW_Release, status:UserConfirmed`，供评委可视化核验。
- **双级去重防线**：
  - L1: Redis SETNX `evt:{tx_hash}:{log_index}` (TTL 24h) 快速过滤。
  - L2: PG `processed_events` 表持久化防重启重复结算。
- **Admin 应急接口鉴权**：`/admin/retry/{jobId}` 必须校验 HTTP Header 中的 `X-Demo-Admin-Token`，防止任意访问强制篡改结算。
### F5: 声誉惩罚与存证
- **链上存证解耦**：合约在 `verifyJobResult(false)` 时自动 `emit JobSlashed`。AEP 后端不主动调用惩罚 API，仅做事件流转发，保持协议独立性。

## 7. 数据流转与可观测性 (零虚构纯落地)
### 7.1 全链路追踪 (纯标准库实现，防协程日志错乱)
- **禁止使用全局变量**存储请求状态，防并发覆盖。
- 每条链上事件/HTTP请求入口，生成 UUID 作为 `TraceID`。
- **所有业务函数、三方API调用强制透传 `Context`**。
- Zap 日志固定格式：`{"trace_id": "xxx", "job_id": "xxx", "tx_hash": "xxx", "event": "CAW_Release", "status": "success"}`，实现秒级问题溯源。
### 7.2 离线回放一致性保障
- 离线回放流使用的 `mock_events.jsonl` 必须通过 `cast` 从测试网抓取真实事件生成，保证数据格式、字段与在线模式完全一致，仅数据源切换，杜绝手写Mock穿帮。

## 8. 风险评估与防御矩阵 (终极版)
| 风险场景 | 严重程度 | 防御与降级策略 (纯落地版) |
|----------|----------|-------------------------|
| **Relayer权限失控** | 致命 | **Buyer确认机制**：结算必须前端确认触发，日志显性打印「用户人工确认，触发合规结算」 |
| **Go协程日志错乱** | 致命 | **Context+TraceID透传**：杜绝全局变量，Zap固定格式打印，全链路溯源 |
| **Redis抢单死锁** | 致命 | **TTL兜底+defer释放**：锁TTL设为2倍任务时长，强制defer解锁，PG行锁二次兜底 |
| **Admin接口越权** | 致命 | **Token鉴权**：所有应急接口增加固定演示Token校验，前端隐藏审核按钮 |
| **金融合规违规** | 致命 | **话术修正+免责声明**：全域采用“密码学锁定调度”，大屏/PPT醒目免责 |
| **LLM格式异常阻塞** | 严重 | **严格降级逻辑**：格式错误/超时直接忽略 LLM，仅依赖规则主轨裁决 |
| **重启导致重复结算** | 严重 | **双级去重防线**：Redis前置快速去重 + PG持久化最终去重 |
| **离线演示数据穿帮** | 严重 | **真实数据回放**：使用Cast抓取真实链上事件生成JSONL，保证数据结构100%一致 |
